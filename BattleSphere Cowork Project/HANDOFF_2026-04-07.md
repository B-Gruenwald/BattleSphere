# BattleSphere – Session Handoff (2026-04-07)

## Project overview
BattleSphere / Warzones is a Next.js 16.2 (App Router) web app for running map-based Warhammer 40K campaigns. Stack: React 19, Tailwind CSS v4, Supabase (Postgres + RLS). Live on Vercel. Source: `/sessions/dreamy-cool-ptolemy/mnt/BattleSphere` — the Supabase project is separate and managed via the dashboard.

Key concept: campaigns have territories (top-level + depth-2 sub-territories + depth-3 landmarks), each with x_pos/y_pos (0–100 SVG coordinate space). Top-level territories are connected by warp routes stored in a `warp_routes` table (campaign_id, territory_a, territory_b).

## Important technical details

**SVG coordinate space**: `viewBox="0 0 100 100"`, `preserveAspectRatio="xMidYMid meet"`. All positions are 0–100.

**`separateNodes(nodes, minDist, ...bounds)`**: force-based repulsion with hard clamp. Sub-territory call uses `(nodes, 5, 80, 2, 98, 2, 96)` — wide bounds intentionally to avoid clamping nodes onto their parent.

**`normalizePositions()`** in CampaignMap: scales/centers top-level territories to fill the canvas, then runs separateNodes for top-level (minDist=12) and sub-territories (minDist=5).

**`createAdminClient()`**: service-role Supabase client that bypasses RLS — used in all admin API routes.

**Warp routes**: normalized so `territory_a < territory_b` (lexicographic). `MapRouteEditor` uses `.sort()` before insert/delete.

**Sub-territory orbits**: depth-2 at radius 9 (x) × 7 (y) from parent; depth-3 at radius 4 × 3.
